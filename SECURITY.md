# Security Policy

## Reporting a Vulnerability

Security is important, even in learning and practice repositories.

If you discover a security vulnerability within this project, please report it responsibly rather than disclosing it publicly. This helps ensure any issues can be addressed without unnecessary exposure.

You can report vulnerabilities by opening a private communication channel (where available) or by submitting a GitHub issue marked clearly as a **security concern**.

Please include:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested mitigation

---

## Supported Scope

This repository is a learning and demonstration project for automation frameworks. It does not operate production systems. However, secure coding practices are still followed to reflect professional engineering standards.

---

## Secure Coding Principles

This project aims to follow:

- No hard-coded credentials
- No sensitive data committed to the repository
- Use of environment variables where appropriate
- Clean dependency management
- Minimal exposure of system-level configuration

---

## Responsible Disclosure

Please do not publicly disclose security issues before they are reviewed. Responsible reporting helps maintain trust and protects users who may reference this repository for learning purposes.

---

Thank you for helping maintain a secure and professional project environment.
