# Contributing Guidelines

Thank you for your interest in contributing to this repository.

This project is a professional learning and automation practice environment focused on Selenium-based testing frameworks. Contributions are welcome where they align with the goals of improving code quality, structure, and automation practices.

---

## Ways to Contribute

You can contribute by:

- Improving test structure and organisation
- Suggesting better automation practices
- Enhancing documentation
- Fixing issues or bugs
- Proposing framework improvements

---

## Before You Contribute

Please ensure:

- Code is clear and readable
- Naming conventions are consistent
- Tests follow the existing project structure
- Changes do not break existing functionality
- Comments explain non-obvious logic

---

## Code Standards

- Use clean, maintainable code
- Avoid unnecessary complexity
- Follow Python and Selenium best practices
- Keep tests independent and repeatable

---

## Pull Request Process

1. Fork the repository
2. Create a new branch with a clear name
3. Make your changes
4. Ensure tests still run successfully
5. Submit a pull request with a clear explanation of your changes

---

## Reporting Issues

If you find a bug or want to suggest an improvement:

- Use the Issues tab
- Provide a clear description
- Include steps to reproduce (if relevant)

---

## Project Goal

This repository exists to demonstrate professional automation engineering practices and framework structure. Contributions should support that goal.

---

Thank you for helping improve the quality and professionalism of this project
